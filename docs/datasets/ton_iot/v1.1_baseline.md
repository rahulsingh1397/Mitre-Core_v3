## Why GMM+BIC works on TON_IoT

The v1.0 investigation diagnosed HDBSCAN over-segmentation: 55 clusters predicted for 10 true classes. GMM+BIC sidesteps the problem entirely:
1. BIC explores k ∈ [2, 20] and selects k=20 deterministically — exactly 2× the true class count
2. GMM assigns ALL points (no noise label −1), so coverage = 1.0
3. Soft assignments via `predict_proba` give meaningful per-point confidence

Cosine_sim on TON_IoT is 0.73 (Exp 2 S1 measurement) — embeddings are healthy and class-discriminative; the v1.0 failure was purely the HDBSCAN k-discovery, not the GNN representation.
