```yaml

  device: cpu
  num_layers: 1
  hdbscan_min_cluster_size: 200
  hdbscan_pca_components: 8
  hdbscan_cluster_selection_epsilon: 0.15
  use_geometric_confidence: true
```

**Sweep rationale:** Default config (mcs=5, pca=16, eps=0.1) produced 45 clusters for 15 true classes
(3× over-segmented due to BENIGN=72.9% imbalance). Sweep winner (mcs=200, eps=0.15) consolidates
BENIGN into ~1 cluster → 11 clusters, close to 15 true classes. ARI improved from 0.111 → 0.177
(+60% relative). Unlike TON_IoT (default config frozen), CICIDS2017 uses the sweep winner because
the default config is geometrically poor. See `docs/datasets/cicids2017/decision_log.md`.

## Root Cause of V3 Underperformance

See `docs/datasets/cicids2017/investigation.md` for full analysis. Summary:

1. **BENIGN over-segmentation**: BENIGN=72.9% of 10K sample (7,290 rows). HDBSCAN mcs=5 finds
   fine-grained density modes within the large BENIGN region → ~30 BENIGN sub-clusters. Default
   config produced 45 total clusters vs 15 true classes.
2. **Sweep partially mitigates**: mcs=200 + eps=0.15 merges BENIGN into ~1 cluster → 11 total.
   ARI improves 0.111 → 0.177. But gap to Spectral(emb) at 0.333 remains.
3. **Spectral (emb) privileged advantage**: Receives exact true class count (n_clusters=15) AND
   operates on V3's own GNN embedding space. Effectively "spectral clustering on V3 embeddings
   with oracle cluster count."
4. **IP-only graph**: No `hostname`/`username` columns → no host/user entity nodes. Only temporal
   edges possible. Same limitation as TON_IoT.
5. **Checkpoint trained on network IDS**: `network_v9_v3` learned on NSL-KDD/UNSW-style traffic;
   CICIDS2017's more complex multi-day capture may produce different embedding geometry.

## Known Limitations

1. **V3 loses on CICIDS2017** — genuine finding. The zero-shot claim holds for network IDS but
   CICIDS2017's extreme class imbalance (BENIGN=72.9%) creates over-segmentation that even the
   sweep winner only partially corrects.
2. **Spectral (emb) unfair advantage**: receives exact true class count AND V3's embeddings. Not
   a realistic deployment baseline; cited as an upper bound on what GNN embeddings can achieve.
3. **3 sparse classes absent from 10K**: Heartbleed (11 rows), SQL Injection (21), Infiltration (36)
   too rare to sample into 10K subset. n_clusters=15 may produce empty clusters for K-Means.
4. **288,602 null rows**: Appear as "UNKNOWN" class in 10K sample (~925 rows, 9.25%). Isolated
   graph nodes (no IP edges). Slight metric penalty — quantified in audit.md.
5. **No CICIDS2017-specific checkpoint**: `network_v9_v3` is a network-IDS checkpoint trained on
   NSL-KDD/UNSW. IoT/DDoS-heavy traffic patterns may not transfer optimally.
